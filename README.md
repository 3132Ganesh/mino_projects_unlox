## What it does

GroupDNA reads a raw WhatsApp chat export (`.txt`), parses it line by line, and produces a formatted analytics report that reveals who in your group is the night owl, the spammer, the ghost, the group mom — and more.

## Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | **Chat Parser** | Reads `.txt` export, extracts timestamp / sender / message. Handles system messages, `<Media omitted>`, deleted messages, and multi-line entries. |
| 2 | **Group Overview** | Headline stats — total messages, date range, per-person message count with bar chart. |
| 3 | **Most Active Day & Hour** | Identifies the single busiest day and peak hour across the full dataset. |
| 4 | **Activity Heatmap** | 6×24 NumPy matrix of messages per person per hour, rendered as a printable block-character heatmap. |
| 5 | **Top Words** | Word frequency across all real messages (stop-words filtered, bar-rendered output). |
| 6 | **Response Patterns & Silent Streaks** | Average response time per person; longest consecutive silent streak. |
| 7 | **Personality Archetypes** | Eight archetype rules — The Spammer, Group Mom, Night Owl, Storyteller, Drama Queen, Ghost, Comedian, Question Master. Every participant gets exactly one. |
| 8 | **Final Report** | Everything above assembled into one clean, formatted, screenshot-worthy report. |

## Constraint list

This project is built entirely within the following boundaries — no exceptions.

**Allowed:**
- Python built-ins: `open()`, `str` methods, `list`, `dict`, `set`, `tuple`, `sorted()`, f-strings, comprehensions
- `numpy` — `np.zeros`, indexing, slicing, aggregations
- `datetime` — only `datetime.strptime` and `timedelta`

**Forbidden (and why):**
- `pandas` — we read and parse the file ourselves with `open()`
- `matplotlib` / `seaborn` / `plotly` — visualization is text-art using block characters (`█ ▒ ░ .`)
- `collections.Counter` / `defaultdict` — word counting is done with a plain `dict`
- `re` (regex) — all parsing uses `str.split()`, `startswith()`, `endswith()`, `isupper()`
- Any pre-built WhatsApp analyzer library

The constraint is the point. Anyone can call `df.groupby()`. The interesting signal is that this was built without it.

## How to run

### Option A — Google Colab (recommended)

1. Open the notebook in [Google Colab](https://colab.research.google.com/).
2. Upload `hostel_bois.txt` using the left sidebar folder icon.
3. Run all cells top to bottom (`Runtime → Run all`).

The file path is set to `/content/hostel_bois.txt` — no changes needed.

### Option B — Local Jupyter

```bash
# Clone the repo
git clone https://github.com/<your-username>/GroupDNA.git
cd GroupDNA

# Install the only dependency
pip install numpy

# Launch
jupyter notebook GroupDNA_Ganesh_23241A67A1.ipynb
```

Place `hostel_bois.txt` in the same directory as the notebook. The path in Cell 2 is `hostel_bois.txt`.

### Running on your own chat

1. Open WhatsApp → any group → More options → Export chat → Without media.
2. Save the `.txt` file as `my_chat.txt` in the same directory.
3. In Cell 2, change `'hostel_bois.txt'` to `'my_chat.txt'`.
4. Run all cells.

> **Privacy note:** Never upload your personal chat to GitHub. Only screenshot the output.

## Dataset

The provided dataset `hostel_bois.txt` is a synthetic WhatsApp export of a fictional engineering hostel group.

| Property | Value |
|----------|-------|
| Group name | Hostel Bois 4ever |
| Date range | 01 April 2024 – 30 May 2024 |
| Total lines | 3,177 |
| Real messages | 3,174 |
| Participants | Rahul, Priya, Aman, Karan, Neha, Vikas |
| Media omitted | ~30 |
| Deleted messages | ~15 |
| System messages | 4 |

Each participant was designed with a deliberate behavioral pattern (Aman really is a night owl at 79.8% after 11 PM; Vikas really is a ghost at 73% silent days). If your archetype detection logic is correct, it will surface the right label for each person — that is the built-in test.

---

## Edge cases handled

| Case | Detection | Action |
|------|-----------|--------|
| System messages | Line has no `sender: message` colon structure | Skipped from analysis; counted separately |
| `<Media omitted>` | Exact string match | Counted as a message for the sender; excluded from word frequency |
| `This message was deleted` | Exact string match | Counted in a per-person deleted-message counter |
| Multi-line messages | Line does not start with `DD/MM/YY` pattern | Appended to the previous message |
| Empty lines | Empty after `.strip()` | Silently skipped |
